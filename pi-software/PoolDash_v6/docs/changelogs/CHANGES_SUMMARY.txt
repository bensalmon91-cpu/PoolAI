================================================================================
WHAT WAS WRONG WITH ALARMS
================================================================================

TECHNICAL DETAILS (for reference):

The alarms API endpoint had this code:

  def parse_meta(row):
      meta = row["meta"]  # ← This column doesn't exist!
      ...

  for r in active:
      active_list.append({
          ...
          "meta": parse_meta(r),  # ← Fails here
      })

The SQL query only selected these columns:
  pool, host, system_name, serial_number, source_label, bit_name, started_ts

There was NO "meta" column in the query results.

When Python tried to access row["meta"], it raised a KeyError and the
entire API endpoint failed, returning an error to the JavaScript.

The fix removed all references to the non-existent "meta" field and
instead uses the actual columns that exist in the query results.
